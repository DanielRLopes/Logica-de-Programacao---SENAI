# 🚀 Projetos Pessoais

Espaço dedicado para **projetos pessoais** e ideias em desenvolvimento.

---

## 📂 Estrutura

| Pasta | Propósito |
|-------|-----------|
| `00-em-ideacao/` | Ideias em brainstorm, ainda não codificadas |
| `01-em-desenvolvimento/` | Projetos ativos atualmente |
| `02-concluidos/` | Projetos finalizados, prontos para portfólio |
| `03-arquivados/` | Projetos pausados ou descontinuados |

---

## 🔄 Fluxo de um Projeto

```
IDEIA → 00-em-ideacao → 01-em-desenvolvimento → 02-concluidos
                              ↓
                        03-arquivados (se pausado)
```

---

## 📝 Como Usar

### 1. Nova Ideia
Crie uma pasta em `00-em-ideacao/` com:
- Nome descritivo (kebab-case)
- README.md explicando a ideia
- Sketches ou notas (opcional)

### 2. Começar Desenvolvimento
Mova para `01-em-desenvolvimento/` e adicione:
- Estrutura do projeto
- README com setup e features
- Código fonte

### 3. Concluir
Mova para `02-concluidos/` e garanta:
- README completo com demo
- Código revisado
- Screenshots ou GIFs (se UI)

### 4. Arquivar
Mova para `03-arquivados/` com:
- README explicando motivo do arquivamento
- Estado atual do projeto

---

## 📋 Templates

### Template de Projeto (README.md)

```markdown
# Nome do Projeto

> Uma linha descrevendo o projeto.

## 🎯 Objetivo
O que este projeto resolve?

## 🛠️ Tecnologias
- Linguagem/Framework
- Bibliotecas principais

## 🚀 Como Rodar
```bash
npm install
npm run dev
```

## 📁 Estrutura
```
projeto/
├── src/
└── README.md
```

## ✅ Status
- [ ] Feature 1
- [ ] Feature 2
```

---

## 💡 Dicas

- Mantenha apenas **1-3 projetos** em `01-em-desenvolvimento/` simultaneamente
- Arquive projetos parados há mais de 3 meses
- Celebre conclusões movendo para `02-concluidos/` 🎉
